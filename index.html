JavaScript Programs

A computer program is a list of instructions to be executed by the computer.
In a programming language, these program instructions are called statements.
JavaScript is a programming language. JavaScript statements are separated by semicolons: 


<p id="add"></p>

JavaScript Statements

JavaScript statements are composed of: Values, Operators, Expressions, Keywords, and Comments.

Values

The JavaScript syntax defines two types of values: fixed values and variable values.
Fixed values are called literals. Variable values are called variables.

Literals

The most important rules for writing fixed values are: Numbers are written with or without decimals: 

10.32482934023, 123, 1.33, 69

Strings are text, written within double or single quotes: 
"Holly", 'Bort', "Ottawa, ON", 'These are strings.'

Variables

In a programming language, variables are used to store data values. JavaScript uses the var keyword to declare variables. An equal sign is used to assign values to variables. In this example, x is defined as a variable. Then, x is assigned (given) the value 10: 
var x;
x = 10;

Operators

JavaScript uses arithmetic operators ( + - * / ) to compute values: 
5 + 10 * 2
JavaScript uses an assignment operator ( = ) to assign values to variables: 
var a, b;
a = 25;
b = 10;

To add those numbers: 
<p id="abvar"></p>

<script>
    var a, b;
    a = 25;
    b = 10;
    document.getElementById('abvar').innerHTML = a + b;
</script>

is the result of the code above.

Expressions

An expression is a combination of values, variables, and operators, which computes to a value.
The computation is called an evaluation. For example, 10 * 10 evaluates to 100: 

<p id="mtply"></p>

<script>
    document.getElementById('mtply').innerHTML = 10 * 10;
</script>

<!-- The script would display the number 100 -->

Evaluations can also contain variables 

<p id="mtplyvar"></p>

<script>
    d = 25;
    document.getElementById('mtplyvar').innerHTML = d * 5;
</script>

<!-- The script would display the number 125 -->


The values can be of various types, such as numbers and strings.
Example, "Jonathan" + " " + "is cool", evaluates to "Jonathan is cool" 

<p id="myName"></p>

<script>
    document.getElementById('myName').innerHTML ="Jonathan" + " " + "is cool";
</script>

<!-- Jonathan is cool -->
<!-- without the + " "  the output would  be Jonathanis cool -->

Keywords

JavaScript keywords are used to identify actions to be performed. The var keyword tells the browser to create variables: 

<p id="varx"></p>

<script>
    var d, f;
    d = 10 + 5;
    f = d * 10;
    document.getElementById('varx').innerHTML = f;
</script>         

<!-- This code returns 150. d which is 15 (10 +5), so f is 15 * 10.-->

Comments

Not all JavaScript statements are "executed". Code after double slashes // or between /* and */ is treated as a comment.
Comments are ignored, and will not be executed: 

<script>//This is a comment</script>


Identifiers

Identifiers are names. In JavaScript, identifiers are used to name variables (and keywords, and functions, and labels).
The rules for legal names are much the same in most programming languages. In JavaScript, the first character must be a letter,
an underscore (_), or a dollar sign ($). Subsequent characters may be letters, digits, underscores, or dollar signs.

Javascript is case sensitive. firstName & firstname are two different variables.

Camel Case: commonly used, starts with a lower case letter, second word begins with capital.
example: firstName, lastName, etc.

The Assignment Operator

In JavaScript, the equal sign (=) is an "assignment" operator, not an "equal to" operator. This is different from algebra.
The following does NOT make sense in algebra: 

x = x + 5
In JavaScript, however, it makes perfect sense: it assigns the value of x + 5 to x.
(It calculates the value of x + 5 and puts the result into x. The value of x is incremented by 5.)

    JavaScript Scope

    In js, objects and functions are also variables.
    In js, scope is the set of variables, objects, and functions you have access to. JavaScript has functino scope: The scope changes inside functions.

    Local JavaScript Variables

    Variables declared withing a JavaScript function, become LOCAL to the function.
    Local variables have local scope: They can only be accessed withing the function. 

    <script>
// code here can not use carName
    function myFunction() {
        var carName = "BMW";
        //code here can use carName
    };
</script>

    Since local variables are only recognized inside their functions, variables with the same name can be used in different functinos.

    Local variables are created when a function starts, and deleted when the functions is completed.

    Global JavaScript Variables

    A variable declared outside a function becomes GLOBAL.
    A global variable has global scope: All scripts and functions on a web page can access it. 

    <script>
    var carName = "BMW";

    // code here can use caeName

    function myFunction() {
        // code here can also use carName
    }
</script>

    JavaScript Switch

    Use the switch statement to select one of many blocks of code to be executed.

    Syntax

    switch (expression) {
    case n:
        code block
        break;
    case n:
        code block
        break;
    default:
        default code block
}


    The switch expression is evaluated once.
    The value of the expression is compared with the values of each case
    If there is a match, the associated block is code is executed.

    Example

    The getDay() method returns the weekday as a number between 0 and 6.
    This example uses the weekday number to calculate the weekday name. 

      var daytoday;
    switch (new Date().getDay()) {
        case 0: 
            daytoday = "Sunday";
            break;
        case 1:
            daytoday = "Monday";
            break;
        case 2: 
            daytoday = "Tuesday";
            break;
        case 3:
            daytoday = "Wednesday";
            break;
        case 4:
            daytoday = "Thursday";
            break;
        case 5:
            daytoday = "Friday";
        case 6:
            daytoday = "Saturday";
            break;
    }
    document.getElementById("todaysdate").innerHTML = "Today is " + daytoday;



    The default Keyword

    The default keyword specifies the code to run if there is no case match: 

    switch (new Date().getDay()) {
    case 6:
        dytext = "Today is Saturday";
        break; 
    case 0:
        dytext = "Today is Sunday";
        break; 
    default: 
        dytext = "Looking forward to the Weekend";
}
document.getElementById("weekend").innerHTML = dytext;

    Different Kinds of Loops

    for loops through a block of code a number of times for/in loops through the properties of an object while loops through a block of code while a specified condition is true do/while also loops through a block of code while a specified condition is true

    For Loop

    The for loop is often the tool used when wanting to create a loop.
    The loop has the following syntax 

    for(statement 1; statement 2; statement 3) {
    code block to be executed
}

    Statement 1 is exeuted before the loop (the code block) starts
    Statement 2 defines the condition for running the loop (the code block)
    Statement 3 is executed each time after the loop (code block) has ben executed 

    <script>
    var text = "";
    var i;
    for (i = 0; i < 5; i++) {
        text+= "The number is " + i + "<br>";
    }
</script>
    Statement 1 sets a variable before the loop starts (var i = 0)
    Statement 2 defines the condition for the loop to run (i must be less than 5)
    Statement 3 increases a value (i++) each time the code block in the loop has been executed

    Statement 1

    Normally you would use statement 1 to initialize the variable used in the loop (i = 0)
    This is not always the case, js doesn't care. Statement 1 is optional
    You can iterate many values in statement 1 separated by commas 

    <p id="automobiles"></p>
            
<script>
    var autos = ["BMW", "Tesla", "Audi", "Kia"];
    var aa, ln, atxt;
    for (aa = 0, ln = autos.length, atxt = ""; aa < ln; aa++) {
        atxt += autos[aa] + "<br>";
    }
    document.getElementById("automobiles").innerHTML = atxt;
</script>

    And you can omit statement 1 (like when your values are set before the loop starts): 

    <p id="automobiles"></p>
            
<script>
    var autos = ["BMW", "Tesla", "Audi", "Kia"];
    var aa = 0;
    var ln = autos.length;
    var atxt = "";
    for (; aa < ln; aa++) {
        atxt += autos[aa] + "<br>";
    }
    document.getElementById("automobiles").innerHTML = atxt;
</script>


    Statement 2

    Often statement 2 is used to evaluate the condition of the initial variable
    This isn't always the case, js doesn't care. Statement 2 is also optional
    If statement 2 returns true, the loop will start over again, if it returns false, the loop will end

    If you omit statement 2, you must provide a break inside the loop. Otherwise the loop will
    never end. This will crash the browser.


    Statement 3

    Often Statement 3 increments the value of the initial variable.
    This ins't always the case, and 3 is optional
    Statement 3 can also do anything like negative increment (i--), positive increment (i = i + 15), or anything else
    Statement three can also be omitted (like when you increment your values inside the loop):
