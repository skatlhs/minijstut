Syntax

JavaScript Programs

A computer program is a list of instructions to be executed by the computer.
In a programming language, these program instructions are called statements.
JavaScript is a programming language. JavaScript statements are separated by semicolons: 


<p id="add"></p>

<script>
    var a, b, c;
    a = 5;
    b = 6;
    c = a + b;
    document.getElementById("add").innerHTML = c;
</script>

(script running in code above)

JavaScript Statements

JavaScript statements are composed of: Values, Operators, Expressions, Keywords, and Comments.

Values

The JavaScript syntax defines two types of values: fixed values and variable values.
Fixed values are called literals. Variable values are called variables.

Literals

The most important rules for writing fixed values are: Numbers are written with or without decimals: 

10.32482934023, 123, 1.33, 69

Strings are text, written within double or single quotes: 
"Holly", 'Bort', "Ottawa, ON", 'These are strings.'

Variables

In a programming language, variables are used to store data values. JavaScript uses the var keyword to declare variables. An equal sign is used to assign values to variables. In this example, x is defined as a variable. Then, x is assigned (given) the value 10: 
var x;
x = 10;

Operators

JavaScript uses arithmetic operators ( + - * / ) to compute values: 
5 + 10 * 2
JavaScript uses an assignment operator ( = ) to assign values to variables: 
var a, b;
a = 25;
b = 10;

To add those numbers: 
<p id="abvar"></p>

<script>
    var a, b;
    a = 25;
    b = 10;
    document.getElementById('abvar').innerHTML = a + b;
</script>

is the result of the code above.

Expressions

An expression is a combination of values, variables, and operators, which computes to a value.
The computation is called an evaluation. For example, 10 * 10 evaluates to 100: 

<p id="mtply"></p>

<script>
    document.getElementById('mtply').innerHTML = 10 * 10;
</script>

<!-- The script would display the number 100 -->

Evaluations can also contain variables 

<p id="mtplyvar"></p>

<script>
    d = 25;
    document.getElementById('mtplyvar').innerHTML = d * 5;
</script>

<!-- The script would display the number 125 -->


The values can be of various types, such as numbers and strings.
Example, "Jonathan" + " " + "is cool", evaluates to "Jonathan is cool" 

<p id="myName"></p>

<script>
    document.getElementById('myName').innerHTML ="Jonathan" + " " + "is cool";
</script>

<!-- Jonathan is cool -->
<!-- without the + " "  the output would  be Jonathanis cool -->

Keywords

JavaScript keywords are used to identify actions to be performed. The var keyword tells the browser to create variables: 

<p id="varx"></p>

<script>
    var d, f;
    d = 10 + 5;
    f = d * 10;
    document.getElementById('varx').innerHTML = f;
</script>         

<!-- This code returns 150. d which is 15 (10 +5), so f is 15 * 10.-->

Comments

Not all JavaScript statements are "executed". Code after double slashes // or between /* and */ is treated as a comment.
Comments are ignored, and will not be executed: 

<script>//This is a comment</script>


Identifiers

Identifiers are names. In JavaScript, identifiers are used to name variables (and keywords, and functions, and labels).
The rules for legal names are much the same in most programming languages. In JavaScript, the first character must be a letter,
an underscore (_), or a dollar sign ($). Subsequent characters may be letters, digits, underscores, or dollar signs.

Javascript is case sensitive. firstName & firstname are two different variables.

Carmel Case: commonly used, starts with a lower case letter, second word begins with capital.
example: firstName, lastName, etc.

Most JavaScript programs contain many JavaScript statements (programs and statements usually called code).
The statements are executed, one by one, in the same order as they are written.
In this example a, b, and c are given values, and finally c is displayed:

The Assignment Operator

In JavaScript, the equal sign (=) is an "assignment" operator, not an "equal to" operator. This is different from algebra.
The following does NOT make sense in algebra: 

x = x + 5
In JavaScript, however, it makes perfect sense: it assigns the value of x + 5 to x.
(It calculates the value of x + 5 and puts the result into x. The value of x is incremented by 5.)

In programming, text values are called text strings.
JavaScript can handle many types of data, but for now, just think of numbers and strings.
Strings are written inside double or single quotes. Numbers are written without quotes.
If you put a number in quotes, it will be treated as a text string.

JavaScript Numbers

Numbers can be written with or without decimals.
Extra large or extra small numbers can be written with scientific (exponential) notation:

var y = 123e5;      // 12300000
var z = 123e-5;     // 0.00123


JavaScript Booleans

Booleans can only have two values, true or false

var x = true;
var y = false;
Booleans are often used in conditional testing.

JavaScript Arrays

JS arrays are written with square brackets, and items separated by commas.
The following creates an array called cars, containing three items.

var cars = ["Tesla", "BMW", "Fiat"]
Arrays indexes are zero-based, meaning the first item is [0], second item is [1], third [2]....

JavaScript Objects

KS objects are written with curly braces { }
Object properties are written as name: value pairs, separated by commas.

var person = {firstName: "Jonathan", lastName: "B", age: 25};
The Person object (var person) has 3 properties: firstName, lastName, and age.

typeof Operator

This can be used to find the type of a js variable.
typeof operator returns the type of a variable or an expression: 

typeof ""                  // Returns "string"
typeof "Jonathan"              // Returns "string"
typeof "Jon" + "athan"      // Returns "string"
typeof 9          // Returns "number"
typeof (14)      // Returns "number"
